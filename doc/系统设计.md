# 系统设计 (System Design) - v2.0 (Implemented)

## 1. 项目概述 (Project Overview)
本系统是一个分布式的爬虫任务管理与监控平台。
系统通过 SSH 协议连接多个独立的计算节点（Node），管理部署在不同服务器上的爬虫任务（Task）。
**核心特色**：
- **无 Agent 模式**：无需在计算节点安装额外 Agent，完全基于 SSH 协议管理。
- **中心节点 (Center Node) 支持**：支持通过跳板机（Jump Host）访问内网节点。
- **实时日志流**：基于 WebSocket 的实时日志监控与分析。
- **自动化健康巡检**：通过分析日志内容自动判断任务状态（运行中/已停止/故障）。

## 2. 总体架构 (Architecture)
采用 B/S (Browser/Server) 架构。

- **前端 (Frontend)**: 
  - 基于 React + TypeScript + Ant Design 构建。
  - 提供节点管理、批量任务创建、实时日志终端 (Xterm.js)、状态仪表盘。
- **后端 (Backend)**: 
  - 基于 Python **FastAPI** 框架。
  - 使用 `asyncssh` 处理高并发 SSH 连接与隧道。
  - 使用 SQLAlchemy + SQLite (可切换) 存储元数据。
- **通信协议**:
  - RESTful API: 元数据管理、命令下发。
  - WebSocket: 实时日志流传输。
  - SSH: 后端与计算节点之间的通信。

## 3. 核心实体与数据模型 (Core Entities)

### 3.1 节点 (Node)
代表一台远程服务器。
- **关键属性**:
  - `host`: 连接地址。
  - `port`: SSH 端口 (可选，默认优先读取 `~/.ssh/config`，否则为 22)。
  - `username`: SSH 用户名 (可选，默认优先读取 `~/.ssh/config`)。
  - `auth`: 支持 SSH Key 或 密码（AES 加密存储）。
  - **`is_center`**: 标记是否为**中心节点** (跳板机/Gateway)。
- **连接逻辑**:
  - **直连模式**: 直接 SSH 连接目标节点。
  - **跳板模式**: 若存在 Center Node 且目标节点非 Center，系统会自动先连接 Center Node，再建立隧道连接目标节点。
  - **凭证继承**: 若目标节点未配置凭证，尝试复用 Center Node 的凭证（方便统一管理内网机器）。

### 3.2 任务 (Task)
代表一个具体的爬虫进程。**Task 与 Node 是 1:1 关系**。
- **关键属性**:
  - `log_file_path`: 远程日志路径 (监控核心)。
  - `start_command` / `stop_command`: 启停命令。
  - `status`: 当前状态 (Running / Stopped / Unknown)。
  - `health_status`: 健康状况 (Healthy / Warning / Error)。
  - `is_enabled`: 启用/禁用状态 (1/0)。**禁用后调度器将跳过该任务的健康检测**。
  - `health_rules`: 关联的健康检查规则。
- **批量创建**:
  - 前端支持一次性选择多个节点，后端会为每个节点创建独立的 Task 记录。

### 3.3 健康规则 (Health Rule)
定义任务健康的判断标准。
- **类型**:
  - `keyword`: 日志出现特定关键词视为异常 (e.g. "Exception", "Captcha Fail")。
  - `latency`: (预留) 包含时延检测。
  - `regex`: (预留) 正则匹配。

## 4. 关键功能模块实现 (Key Modules)

### 4.1 节点连接管理器 (SSH Manager)
`backend/app/services/ssh_service.py`
- 封装 `asyncssh`。
- **自动隧道管理**: 在连接非中心节点时，自动检测并复用中心节点连接作为 Tunnel。
- **连接测试**: 提供 `/test-connection` 接口，验证直连或跳板连接的连通性。

### 4.2 系统配置管理 (System Settings)
`backend/app/models/system.py`
- **存储方式**: 使用数据库表 `system_settings` 存储系统级配置 (替代原本的 config.json)。
- **动态更新**: 支持通过 API 动态修改配置 (如 `inspection_interval`) 并实时生效。

### 4.3 日志监控与分析 (Log Monitor & Analysis)
`backend/app/services/log_analysis.py`
- **实时流模式**: 
  - 通过 WebSocket (`/ws/tasks/{id}/log_stream`) 建立连接。
  - 后端执行 `tail -f` 并将 stdout 实时推送到前端。
- **实时分析逻辑**:
  - 在流传输过程中，缓冲并分析日志行：
  1. **状态判定**: 解析日志最新时间戳。若最后更新时间即使当前时间 > 1小时，标记为 `STOPPED`。
  2. **错误率检测**: 若最近日志片段中 Error 行占比 > 40%，标记 `health_status` 为 `ERROR`。
  3. **关键词检测**: 匹配用户配置的 `HealthRule`，命中即报错。
  4. **状态回写**: 分析结果实时更新到数据库 (`last_log_time`, `status`, `health_status`)。

### 4.4 任务控制 (Task Control)
- **Start/Stop/Restart**:
  - 后端通过 SSH 发送 shell 命令。
  - 若配置了中心节点，命令会自动通过跳板机路由。
  - **乐观更新**: 发送命令成功后立即更新数据库状态为 Running/Stopped。
- **Edit**:
  - 支持修改任务配置（日志路径、健康规则等）。

## 5. 接口设计 (API Design)

| Method | Path | Description |
|---|---|---|
| **Nodes** | | |
| GET | `/api/nodes/` | 获取节点列表 |
| POST | `/api/nodes/` | 创建节点 (支持设置中心节点) |
| POST | `/api/nodes/test-connection` | 测试连接 (支持跳板测试) |
| **Tasks** | | |
| GET | `/api/tasks/` | 获取任务列表 |
| POST | `/api/tasks/` | **批量创建任务** (传入 `node_ids` 列表) |
| WS | `/api/tasks/{id}/log_stream` | WebSocket 日志流 + 实时分析 |
| POST | `/api/tasks/{id}/action` | 执行 start/stop/restart |
| PUT | `/api/tasks/{id}` | **编辑任务** (支持修改配置) |
| **System** | | |
| GET/POST | `/api/system/settings` | 获取/更新系统配置 |

## 6. 技术细节与配置
- **SSH 配置**: 后端优先读取宿主机 `~/.ssh/config`，实现免密登录、别名管理和端口控制。
- **数据库**: 使用 SQLAlchemy ORM，可轻松迁移至 MySQL/PG。
- **安全性**: 
  - 密码字段使用 Fernet (symmetric encryption) 存储。
  - SSH 连接使用 KnownHosts (可选宽松模式)。
